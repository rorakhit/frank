Self-hosted personal finance automation. Connects to your bank accounts via Plaid, categorizes transactions with Claude AI, and surfaces spending insights through a built-in web dashboard, Gmail alerts, and optional Notion reports.

## Features

- **Real-time transaction ingestion** via Plaid webhooks with JWT verification
- **AI categorization** using Claude — with 90-day merchant history for context
- **Web dashboard** — Overview, Credit Health, Savings Plan, Reports, Flagged Transactions, Recent Transactions
- **Export** — CSV and JSON export for all tabular views; print-to-PDF for report pages
- **Gmail alerts** — large purchases, duplicate charges, new subscriptions, daily spend, credit thresholds
- **Paycheck detection** — triggers a biweekly spend summary + savings recommendation email
- **Monthly/yearly reports** — auto-generated by cron on the 1st and Jan 1
- **Notion dashboard** — optional; all 7 Notion pages are skipped if `NOTION_TOKEN` is not set
- **Credit tracking** — per-card utilization, APR, monthly interest, avalanche payoff order
- **Loan tracking** — balance, principal paid YTD, estimated payoff

## How It Works

```
Bank transaction
      │
      ▼
Plaid webhook ──► JWT signature verification
      │
      ▼
Claude AI categorizes transaction
(with 90-day merchant history context)
      │
      ├──► Supabase (PostgreSQL)
      │
      ├──► Web dashboard (/dashboard)
      │
      ├──► Real-time Gmail alerts
      │         • Large purchase > $200
      │         • Duplicate charge detected
      │         • New subscription
      │         • Daily spend > $300
      │         • Credit utilization ≥ 30% / 50%
      │
      └──► Paycheck detected?
                │
                ▼
          Biweekly spend summary + AI savings recommendation
          → Gmail email
          → Notion dashboard update (optional)

Monthly/yearly cron → reports stored in DB + optional Notion pages
```

> **Paycheck webhook timing:** Plaid fires `SYNC_UPDATES_AVAILABLE` within 1–2 hours of a direct deposit for major banks, but can take 6–12 hours (standard polling cadence) or up to 24–48 hours for smaller banks and credit unions. If the paycheck email doesn't arrive, use the **Run report now** button on `/paycheck` to trigger it manually.

## Stack

- **Runtime:** TypeScript + Fastify 5
- **Database:** Supabase (PostgreSQL) with Row Level Security
- **Bank data:** Plaid API (webhook-driven, cursor-based sync)
- **AI:** Claude API (`claude-haiku-4-5-20251001`) — categorization, narratives, alert enrichment
- **Dashboard:** Built-in web UI (no Notion required)
- **Alerts:** Gmail via nodemailer
- **Deploy:** Docker / Railway / Render

## Quick Start

### Prerequisites

- [Plaid account](https://dashboard.plaid.com/signup) — sandbox is free, no bank approval needed for testing
- [Supabase project](https://supabase.com) — free tier is sufficient
- Node 20+ **or** Docker

### 1. Clone and configure

```bash
git clone https://github.com/rorakhit/autobudget-oss.git
cd autobudget-oss
cp .env.example .env
# Fill in PLAID_CLIENT_ID, PLAID_SECRET, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, SETUP_SECRET
```

### 2. Set up the database

Run the migration files in order in the Supabase SQL editor:

```
supabase/migrations/001_initial.sql
supabase/migrations/002_rls.sql
supabase/migrations/003_categorization_rules.sql
supabase/migrations/004_custom_categories_display_name.sql
supabase/migrations/005_loan_accounts.sql
```

### 3. Run with Docker

```bash
docker compose up
```

The app is available at `http://localhost:3000`.

### 3 (alternative). Run with Node

```bash
npm install
npm run dev   # hot reload via tsx watch
```

### 4. Connect bank accounts

Visit `http://localhost:3000` and enter your `SETUP_SECRET` to authenticate.

Then go to `/link` to connect your bank accounts via Plaid Link.

### 5. Set up credit card details (optional)

Visit `/setup` to enter APRs and a savings goal. This enables the credit health and savings plan features.

### 6. Register the Plaid webhook

In the [Plaid dashboard](https://dashboard.plaid.com), set your webhook URL to:

```
https://<your-deployed-url>/webhook
```

For local testing, use [ngrok](https://ngrok.com): `ngrok http 3000`, then set `PLAID_WEBHOOK_URL` in `.env`.

### 7. Trigger an initial sync

```bash
curl -X POST "http://localhost:3000/link/sync-all" \
  -H "Cookie: ab_auth=<your-SETUP_SECRET>"
```

Or visit the app and click **Sync** on the homepage.

## Deploy to Railway

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/new/template)

1. Fork this repo
2. Create a new Railway project from your fork
3. Add all environment variables from `.env.example`
4. Set `PLAID_WEBHOOK_URL` to your Railway URL + `/webhook`

Railway auto-deploys on push. The `railway.toml` is already configured.

## Deploy to Render

1. Fork this repo
2. New Web Service → connect your fork → select **Docker** environment
3. Add environment variables
4. Deploy

## Optional: Notion

If you want Notion dashboard pages in addition to the web dashboard, set `NOTION_TOKEN` and `NOTION_ROOT_PAGE_ID`. When these are set, AutoBudget writes 7 pages to your Notion workspace after each sync/report. See `.env.example` for details.

---

## Optional: AI categorization

Set `ANTHROPIC_API_KEY` to enable Claude-powered transaction categorization. Without it, transactions are assigned to the `Uncategorized` category and the AI narrative in reports is skipped.

---

## Optional: Gmail alerts

Set `GMAIL_USER` and `GMAIL_APP_PASSWORD` to receive real-time alerts and paycheck reports by email. Generate an app password at [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords).

Set `GMAIL_RECIPIENT` if you want alerts sent to a different address than `GMAIL_USER`.

## Admin Pages

All pages require authentication (session cookie set at login).

| Page | Purpose |
|------|---------|
| `/link` | Connect bank accounts via Plaid Link |
| `/setup` | Enter credit card APRs and savings goal |
| `/review` | Review and correct transaction categories |
| `/rules` | Manage auto-categorization rules |
| `/settings` | Rename accounts, manage custom categories |
| `/paycheck` | Configure paycheck account and recurring allocations |
| `/apple-card` | Import Apple Card CSV transactions |

## Debugging Endpoints

All require authentication.

| Endpoint | Purpose |
|----------|---------|
| `POST /link/repair-webhooks` | Re-register webhook URL on all Plaid items |
| `POST /link/sync-all` | Manually trigger a full sync for all items |
| `POST /link/refresh-notion` | Rewrite Notion pages from DB (no-op if Notion not configured) |
| `GET /link/accounts` | List all linked institutions and accounts |
| `GET /health` | Returns `{ status: "ok" }` |

## Security

- Plaid webhooks verified via JWT signature before any processing
- All app pages and API routes gated by `SETUP_SECRET` cookie
- Supabase service role key used server-side only; RLS blocks all anon access
- All credentials in environment variables — nothing in source code
